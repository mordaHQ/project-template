# 📁 Структура проекта

Проект организован по принципам чистого кода, модульности и предсказуемости.  
Каждая папка имеет чёткое назначение, что ускоряет разработку и помогает избегать путаницы.

---

## 🗂 Общий обзор
project-template/ │ ├── src/                     → исходный код проекта ├── tests/                   → модульные тесты ├── dist/                    → скомпилированный build (автогенерация) ├── scripts/                 → служебные скрипты для CI/CD ├── assets/                  → статические ресурсы ├── markdown-page/           → markdown-примеры для Docusaurus ├── docusaurus-build/        → документация (отдельный сайт) └── ...

---

## 📦 src/ — основной код проекта

Содержит всю бизнес-логику, инфраструктуру и API.

src/ │ ├── domain/              → сущности, value-objects, бизнес-логика ├── application/         → use-cases, сервисы приложения ├── infrastructure/      → API, базы данных, логирование, интеграции ├── presentation/        → API endpoints, CLI, UI-адаптеры ├── utils/               → вспомогательные функции └── index.ts             → входная точка проекта

### Особенности:
- строгое разделение ответственности  
- целостное соблюдение Clean Architecture  
- лёгкость навигации  

---
## 🧪 tests/ — тесты

tests/ │ ├── unit/                → модульные тесты ├── integration/         → интеграционные тесты └── mocks/               → заглушки и фэйковые имплементации

Тесты независимы, быстрые и повторяемые.

---

## 🛠 scripts/ — утилиты и автоматизация

scripts/ │ ├── release.ts           → автоматизация релизов ├── validate.ts          → линтинг/валидация └── setup.ts             → инициализация проекта

Используется GitHub Actions, Husky, lint-staged.

---

## 🗂 assets/ — статические ресурсы

assets/ │ ├── icons/ ├── logos/ └── images/

Используется для документации, UI и предпросмотров.

---

## 📚 docusaurus-build/ — документация

Это отдельный Docusaurus-проект, который публикуется в GitHub Pages.

Ключевые директории:

docusaurus-build/ │ ├── docs/                → твоя документация ├── blog/                → блог-посты ├── src/                 → компоненты React, стили └── build/               → финальный сайт (авто)

---
## 📝 Дополнительные файлы

package.json             → зависимости, скрипты tsconfig.json            → конфигурация TypeScript .editorconfig            → единый стиль кода .gitignore               → исключения для Git .prettierrc              → правила форматирования .eslint.config.mjs       → линтинг проекта

---

## 🚀 Принципы организации структуры

- Минимальная связность слоёв
- Ясные границы ответственности
- Логика отделена от инфраструктуры
- Файлы легко найти без документации
- Проект масштабируется без хаоса

---

## 🎯 Итог

Такая структура:
- ускоряет разработку  
- упрощает онбординг новых разработчиков  
- делает проект поддерживаемым годами  
- создаёт базу для CI/CD, релизов и документации  

Это фундамент production-ready экосистемы, которую мы строим.
