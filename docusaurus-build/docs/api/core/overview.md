# API Overview

Этот раздел описывает общие принципы API, структуру маршрутов, стандарты запросов и ответов, а также правила авторизации.

---

## Базовый стиль API

Проект использует:

- REST архитектуру  
- JSON в качестве формата данных  
- Единый стандарт ошибок  
- Версионирование маршрутов через `/api/v1/`  
- Авторизацию через Bearer Token (JWT)
